# Versus-Airlines :: Style Standard

This is the binding style standard for every Luau file in this project. The
intent: a Versus-Airlines file should be immediately recognizable as
**ours**, should be readable in isolation, and should look hand-written by
a senior engineer — not pasted together from upstream hubs.

The standard is a **checkable** spec. `verify.py` (in this directory)
runs a series of grep-based rules and exits non-zero on any violation.

## 1. File header (every `.lua` file)

Every file starts with a `--[[ ... ]]` block of this shape:

```lua
--[[
    Versus-Airlines :: <module/path>
    ---------------------------------
    <one-sentence purpose>

    Public API:
        <Name>.<method>(<args>) -> <return>
        <Name>.<method>(<args>) -> <return>
]]

local <Name> = {}
<Name>.__index = <Name>
```

Rules:
- The first line is always `Versus-Airlines :: <module/path>`.
- The dashes under it match the length of the title line exactly.
- The "Public API" section lists every public function or method. If the
  file has no public API (e.g. a constants-only file), the section says
  so explicitly: `Public API: none (constants only)`.
- No `discord.gg/...` lines. No `Generated by ...` lines. No
  `Thanks to ...` shoutouts. Attribution goes in the commit message,
  not the source.

## 2. Local names

Allowed:
- Long descriptive names: `monsterList`, `webhookUrl`, `currentSection`.
- Standard iterator names: `i, v`, `k, v` inside `for ... in` loops.
- Loop counters: `i, j, k, n` in tight numeric loops.
- Single-letter locals for: `x` for the first positional argument in a
  generic `wrap` function, `s` for a string parameter, `t` for a table
  parameter.

Disallowed in any non-temp scope:
- Cryptic one-letter names that don't fit the above: `u6, u7, u8, a, q`.
- Names from copy-paste like `v139`, `aaa`, `tmp1`, `temp2`.
- Names containing digits unless they reference a known index.

## 3. Indentation and whitespace

- **4 spaces.** No tabs. (Tabs are checked and rejected by `verify.py`.)
- One blank line between top-level functions, two blank lines between
  top-level sections.
- No trailing whitespace on any line. `verify.py` greps for `[ \t]$`.

## 4. Section dividers

Use a single style for section dividers, length matched to the section
name:

```lua
-- ============================================================================
-- Section name
-- ============================================================================
```

Or for a sub-section:

```lua
-- ----- Section name -----
```

Never use `=============` without `--` prefix. Never use `-- ----- -----` or
other run-on patterns.

## 5. Magic numbers

Every magic number is named at the top of the file in a "Constants" block:

```lua
local MIN_THROTTLE_MS      = 100
local MAX_THROTTLE_MS      = 300
local DEFAULT_THROTTLE_MS  = 200
local WEBHOOK_MIN_SPACING  = 1.0
local UPDATE_INTERVAL      = 2.0
```

The block is placed between the module docstring and the `local X = {}`
declaration. `verify.py` flags any number literal in a function body
that's also a candidate for naming (heuristic: any float with > 1
decimal place).

## 6. Stringly-typed config

Anywhere a function takes a "level", "type", or "kind" argument that has
a fixed set of valid values, define an enum-like table:

```lua
local LogLevel = {
    INFO  = "info",
    WARN  = "warn",
    ERROR = "error",
}
```

Then accept `LogLevel.INFO` etc. — never `"info"` as a bare string. The
`verify.py` script flags any call site that passes a bare string to a
known-typed function.

## 7. Error handling policy

Every public function is `pcall`-guarded at the call site (in the
caller) or wraps its body in `pcall` (in the callee). The policy is
documented in the file header:

```lua
--[[
    ...
    Error policy: every public method is pcall-guarded. Internal helpers
    are not; an internal helper that throws means the caller is broken.
]]
```

## 8. Constants vs. magic strings

`0.016` (the heartbeat interval), `2.0` (the status update interval),
`0.25` (the log throttle), `1.0` (the webhook spacing) — every one of
these gets a name. The list is enforced by `verify.py`.

## 9. Connection management

Every `:Connect` call goes through `kernel.conn:track(conn, tag)`. There
is no `:Connect` outside of a `track` wrapper, except inside the
`kernel/connection.lua` file itself. `verify.py` greps the source tree
for `:Connect(` and flags any match outside the kernel dir.

## 10. Comments

- Every public function has a 1–3 line `--` comment immediately above it
  describing what it does, what it returns, and what it does NOT do.
- Every non-obvious decision has a `-- NOTE: ...` comment explaining
  the why. The why is more important than the what.
- No `TODO` comments left in committed code. TODOs go in
  `docs/TODO.md` if they're real, or get fixed before commit.

## 11. UI mounts (Versus library only)

Every section in `src/ui/sections.lua` follows this template:

```lua
local <name> = ui:CreateSection("<Display Name>")
<name>:createToggle({
    Name     = "<Display Name>",
    Flag     = <default bool>,
    flagName = "<snake_case_flag>",
    Callback = function(enabled) <body> end,
})
```

`flagName` is always `snake_case` and unique across the file. `verify.py`
flags duplicates.

## 12. No dead code

No commented-out blocks. No `local _ = function() end` placeholders.
No `print("here")` debug statements. The smoke test (`verify_slice1.py`
and its successors) is the place for "did this work" output, not the
source.

## 13. Header block on `loader.lua`

`loader.lua` is the public face of the project. Its docstring includes:

```lua
--[[
    Versus-Airlines :: loader
    --------------------------
    Public entry point. Loaded via:
        loadstring(game:HttpGet("<host>/loader.lua"))()

    Or pasted into an executor and run directly. Either way, this file
    is the only thing the user sees. It:

        1. Loads the Versus UI library (src/ui/window.lua)
        2. Boots the kernel (src/kernel/init.lua)
        3. Mounts the section tree (src/ui/sections.lua)
        4. Starts the live status label (src/ui/status.lua)
        5. Returns the kernel graph for the user to interact with

    Version: <semver> | <one-line description of slice>
    Repository: <url>
    License: <license>
    Entry point: <host>/loader.lua
]]
```

The `<host>` placeholder is filled in at deploy time, not now.
